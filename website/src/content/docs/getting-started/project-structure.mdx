# Project Structure

A RustScript project has a minimal layout. Two files to start.

## Layout

```
my-project/
  cargo.toml          # Package metadata
  src/
    index.rts          # Entry point
```

After building, the compiler generates a `.rsc-build/` directory:

```
my-project/
  cargo.toml
  src/
    index.rts
  .rsc-build/          # Generated -- don't edit
    Cargo.toml         # Full Cargo manifest with detected dependencies
    src/
      main.rs          # Generated Rust code
    target/            # Cargo's build artifacts
```

## cargo.toml

The minimal `cargo.toml` you provide:

```toml
[package]
name = "my-project"
edition = "2021"
```

That's it. The compiler reads this for the package name and edition, then generates a full `Cargo.toml` inside `.rsc-build/` with all detected dependencies.

## src/index.rts

Your entry point. Must contain a `function main()` (or `async function main()` for async programs).

```typescript
function main() {
  console.log("Hello!");
}
```

You can organize code into multiple `.rts` files, but `index.rts` is where execution starts.

## .rsc-build/

The compiler's output directory. Contains:

- **Cargo.toml** -- A complete manifest with `[dependencies]` auto-detected from your imports. If you write `import { Router } from "axum"`, the compiler adds `axum` to the generated Cargo.toml.
- **src/main.rs** -- Your RustScript code compiled to Rust. This is valid, human-readable Rust.
- **target/** -- Standard Cargo build artifacts (the compiled binary lives here).

Add `.rsc-build/` to your `.gitignore`. It's a build artifact.

## Templates

`rsc init` supports templates for common project types:

```bash
rsc init my-api --template web-server    # axum web server scaffold
rsc init my-cli --template cli           # CLI tool scaffold
rsc init my-wasm --template wasm         # WASM project scaffold
```

## The compilation pipeline

```
.rts  -->  Parse  -->  AST  -->  Type Check  -->  Lower  -->  Rust IR  -->  Emit  -->  .rs  -->  cargo build  -->  binary
```

Each stage:

1. **Parse** -- Lexer + recursive descent parser turns `.rts` into a RustScript AST
2. **Type Check** -- Resolves types, checks assignments, infers where possible
3. **Lower** -- Transforms the RustScript AST to Rust IR. This is where TypeScript patterns become Rust patterns: `Array<T>` becomes `Vec<T>`, `console.log` becomes `println!`, ownership is inferred
4. **Emit** -- Rust IR becomes `.rs` text
5. **Cargo** -- Standard `cargo build` compiles the generated Rust to a native binary

## The eject option

Because the generated Rust is idiomatic and human-readable, you can eject at any time. Copy the `.rs` files out of `.rsc-build/`, set up your own Cargo project, and continue in pure Rust. No lock-in.
