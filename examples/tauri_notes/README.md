# Tauri Notes

A desktop notes app with a RustScript backend and React frontend, connected via Tauri IPC. This example compiles and runs end-to-end.

## What it demonstrates

- `@command` decorator — compiles to `#[tauri::command]` via import-aware resolution
- `derives Serialize, Deserialize` — shared types between backend and frontend
- `rust { }` escape hatch — three lines of Rust for the Tauri bootstrap macro
- `rsc types` — generates `.d.ts` files so the React frontend shares types with the backend
- In-place compilation — `build.rs`, `tauri.conf.json`, `frontend/` all just work
- Cargo.toml merge — manually added `[build-dependencies]` preserved across builds

## Architecture

```
tauri_notes/
  rustscript.json          # RustScript manifest
  Cargo.toml               # Generated + committed (with manual [build-dependencies])
  build.rs                 # Tauri build script
  tauri.conf.json          # Tauri app configuration
  src/
    main.rts               # RustScript backend: commands, types, business logic
  frontend/
    src/App.tsx             # React frontend calling invoke() over Tauri IPC
    src/types/index.d.ts    # Generated by rsc types — shared with backend
```

## The `@command` Decorator

```typescript
import { command } from "tauri";

@command
function get_notes(): Array<Note> {
  // callable from frontend via invoke('get_notes')
}
```

Compiles to `#[tauri::command] fn get_notes() -> Vec<Note>`.

## Run

```bash
cd frontend && npm install && npx vite build && cd ..
rsc build
cargo run
```
