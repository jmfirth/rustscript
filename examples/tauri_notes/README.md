# Tauri Notes — RustScript Desktop App Example

A notes app demonstrating RustScript as the backend for a Tauri desktop application, using `@command` decorators for IPC.

## Architecture

- **Backend**: RustScript with `@command` decorators (compiles to Rust via `rsc build`)
- **Frontend**: React + TypeScript
- **Shared types**: Generated by `rsc types -o frontend/src/types/`

## The `@command` Decorator

RustScript supports decorators that compile to Rust attributes. When you import `command` from `"tauri"`, the `@command` decorator compiles to `#[tauri::command]`:

```typescript
import { command } from "tauri";

@command
function get_notes(): Array<Note> {
  // ...
}
```

Compiles to:

```rust
#[tauri::command]
fn get_notes() -> Vec<Note> {
    // ...
}
```

This makes the function callable from the frontend via Tauri's `invoke()` IPC.

## Shared Types via `rsc types`

The backend defines types in RustScript:

```typescript
type Note = {
  id: u32,
  title: string,
  content: string,
  pinned: bool,
} derives Serialize, Deserialize
```

`rsc types` generates TypeScript definitions the frontend imports:

```typescript
import type { Note } from './types';
```

One source of truth. Change the backend type, regenerate, and the frontend gets compile errors on any mismatch.

## Building

```bash
# Build the RustScript backend and generate frontend types
rsc build --emit-types frontend/src/types/

# Or generate types separately
rsc types -o frontend/src/types/

# Build the frontend
cd frontend && npm run build
```

## Note on Tauri Builder Setup

The `main()` function in this example runs the commands directly to demonstrate they work. In a real Tauri app, you would wire up the Tauri Builder with `generate_handler![]`, which requires a `rust { }` block or manual Rust glue:

```rust
fn main() {
    tauri::Builder::default()
        .invoke_handler(tauri::generate_handler![get_notes, get_stats, search_notes])
        .run(tauri::generate_context!())
        .expect("error running tauri app");
}
```

The key value is in the `@command` decorator pattern and shared types — the Tauri Builder setup is standard Rust boilerplate.
